Icebreaker is an efficient, lightweight **data lakehouse** that eliminates wasted compute. 

## Icebreaker Objectives

Icebreaker product objectives are to give enterprises:
1. An **affordable serverless experience** that's incredibly convenient for users
2. That a central data team can control in a **private cloud service**
3. And that enforces **shared accountability** to align data budgets with business value.


By delivering on these objectives with Icebreaker, data teams can: 

- Control and defend their data cloud **budgets**
- Meet **SLAs** (performance, availability, and capacity goals)
- Deliver **multi-tenant data products** that meet data privacy, security, and sovereignty requirements

## Key Capabilities

Icebreaker is different because **Workload Optimization** capabilities are built into the data engine. 

With Icebreaker, you can easily:

- Minimize **idle compute** to maximize waste reduction 
- **Rightsize** every query to balance resource consumption with business value delivery
- **Isolate** every query for privacy and security
- Centralize technical management for **economies of scale**
- Distribute financial **accountability** for responsible budgeting and usage

Learn more about how the FinOps Foundation Framework presents [Workload Optimization](https://www.finops.org/framework/capabilities/workload-optimization/) as part of the Optimize Usage & Cost scope.

## Common Usage Patterns

Only Icebreaker frees up data teams and cloud resources to safely and efficiently: 

- Deliver a private, centrally controlled **SQL as a Service user experience** to internal and external customers
- **Smooth demand peaks** for better  user experiences
- Embed analytic functionality into **multi-tenant** and SaaS applications
- Simplify **data sovereignty** compliance
- Confidently deploy **Agentic AI** to respond to unpredictable chatbot queries
